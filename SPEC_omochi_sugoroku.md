# オモチの まなび ぼうけん ─ すごろく版 設計仕様書

> Claude Code への指示書。これを渡して実装してもらう。
> 既存の `omochi_manabi.html`（計算・社会・理科の3教科クイズ）を土台に、
> すごろく風の冒険ゲームへ拡張する。

---

## 0. この仕様の最優先事項（絶対にブレさせない）

**初見の3秒で「やってみたい」と思わせること。**
理由：対象の子ども（小4・息子）は「初見がしょぼいと二度と触らない」性格。
したがって、開いた瞬間の演出 ＞ 中身の作り込み、という優先順位を守る。

作り込みの優先順位（上から重要）:
1. オープニング演出（開いた瞬間のインパクト）
2. キャラが生きてる感（動く・表情が変わる・反応する）
3. すごろくで「冒険してる・進んでる」感
4. ボス／エフェクト／BGM などの飾り

→ 4を頑張る前に、必ず1がしっかり刺さる状態にする。

---

## 1. ゲーム全体の流れ

```
[オープニング] → [マップ画面] → [マスに止まる] → [クイズ] → [結果でコマが進む]
        ↑                                                         │
        └──────────── ゴール到達でクリア演出 ←──────────────────────┘
```

- 1教科＝1つの「すごろくマップ（ステージ）」。計算・社会・理科の3マップ。
- 各マップは 8〜10マス。スタート → ゴールへコマ（オモチ）が進む。
- マスに止まる → そのマスのクイズが1問出る → 正解で前進、不正解はその場（または1マス分ゆっくり）。
- ゴール到達でステージクリア演出。メダル集計。

---

## 2. 画面ごとの仕様

### 2-1. オープニング画面（最重要）
- 開いた瞬間に自動再生される短い演出（2〜4秒）。
- タイトル「オモチの まなび ぼうけん」がアニメーションで登場（ズームイン or 文字が一つずつ出る）。
- オモチ（ハムスター）が歩いて／ジャンプして登場、こちらを見て手を振る等。
- 背景に歯車がゆっくり回る、星や光の粒がふわふわ漂う（スチームパンク×やさしい世界観）。
- 効果音が出せるなら、登場時に「ポンッ」など軽い音（※音は最初ミュート可。ボタンでON）。
- 「▶ ぼうけんスタート」ボタンを大きく1つ。押すとマップ選択へ。

### 2-2. マップ選択画面
- 3つのマップを世界として見せる：
  - 🔢 けいさんの国（ネオン×数字の世界）
  - 🗾 しゃかいの国（地図・日本・岡山モチーフ）
  - 🔬 りかの国（実験・星空モチーフ）
- 各マップに「メダル ○/10」の進捗が出る（クリア度が見える）。

### 2-3. マップ（すごろく）画面
- 曲がりくねった道に丸いマスが並ぶ。スタート旗とゴール旗。
- オモチのコマが現在地に立っている。
- マスの種類（最初はシンプルでよい。余裕があれば追加）:
  - ★ふつうのマス：クイズ1問
  - ？ボーナスマス：正解でメダル多め
  - （将来）ワープ・休みマスなど
- 「すすむ」ボタン or マスをタップ → そのマスのクイズへ。
- コマが進むとき、1マスずつ「ぴょん、ぴょん」と移動アニメ。

### 2-4. クイズ画面
- 既存の3教科の問題をそのまま流用（計算は自動生成、社会・理科は問題プール）。
- オモチが画面にいて、正解で喜ぶ／不正解でしょんぼり、の表情変化。
- 正解：コマが前進。星やコインが飛ぶエフェクト。
- 不正解：正解を表示（「こたえ：○○」）。コマは進まない or 半歩。ライフは設けず、何度でも前向きに進める設計（小4が心折れないように）。

### 2-5. ゴール／クリア画面
- ゴール旗にオモチが到達 → 旗を立てる／ジャンプして喜ぶ大きめ演出。
- 集めたメダル数、星評価（⭐の数）。
- 「もういちど」「べつのマップへ」ボタン。

## 3. キャラクター（オモチ）の扱い ★ザイの担当領域

- 現状は絵文字 🐹。これを **Omochiのスチームパンク・ゴールデンハムスターの画像**に差し替える。
- 画像はザイが Gemini Nano Banana で用意する。理想は表情ちがいを複数：
  - ふつう（待機）
  - 笑顔（正解）
  - 考え中（出題時）
  - しょんぼり（不正解）
  - 大喜び（クリア）
- 透過PNG推奨。サイズ感は揃える。
- **画像が用意できるまでは、絵文字 🐹 のままで実装を進めてOK。**
  あとから画像パスを差し替えるだけで切り替わるよう、
  キャラ表示部分を「画像 or 絵文字」を1か所で切り替えられる構造にしておくこと。

## 4. 技術的な条件（Claude Codeへ）

- **公開方法は GitHub Pages を維持**（既存リポジトリ tititi533551-create / omochi-manabi）。
  スマホのSafari/ChromeでURLを開くだけで遊べ、ホーム画面追加できること。
- 既存の「ホーム画面追加でアプリっぽく開く」メタタグ設定は維持・継承する。
- 画像など複数ファイルになってもよいが、**できるだけ軽量**に（子のスマホで一瞬で開くこと）。
- ブラウザのlocalStorage等で「どこまで進んだか」を保存できると尚良い（次回続きから）。
  ※ただし最初の実装では必須ではない。まず初見フックを優先。
- アニメーションは CSS / JS の軽量なもので可。重い3Dや大きなライブラリは避ける。
- 音を入れる場合、初期はミュート、子がボタンでONにできる形に（勝手に音が鳴ると驚いて閉じる対策）。

## 6. 段階リリースの考え方（重要：一気に全部作らない）

初見フックを最短で確かめるため、以下の順で区切って作る。
各段階で一度「動くURL」にして、子の反応を見られる状態にする。

- **フェーズA（最優先）**：オープニング演出 ＋ マップ1本（計算の国）＋ コマが進む ＋ クリア演出。
  → これだけで「初見で掴めるか」を息子で検証できる。
  - 卒業条件：息子が初見で「やってみる」と触り、最後まで1マップ進んだ。
  - 撤退条件：演出を見ても触らない／途中でやめた → 演出の方向性を作り直す（盛り方が違う）。
- **フェーズB**：社会の国・理科の国を追加（3マップ揃える）。
- **フェーズC**：ボーナスマス、効果音、進捗セーブ、ボス演出など飾りを追加。

※ オモチ画像の差し替えは、フェーズA〜Cどこでも、画像が用意でき次第いつでも。

## 7. ザイ ⇄ Claude Code ⇄ Claude(相談) の分担

- **ザイ**：オモチ画像の用意（Gemini Nano Banana）。完成URLを自分のiPhone/息子のAndroidに入れる。各フェーズで息子の反応を観察し、次の判断材料にする。
- **Claude Code**：本仕様に沿った実装すべて（演出・マップ・アニメ・公開）。
- **Claude(チャット)**：仕様の調整、息子の反応に応じた次フェーズの指示書づくり、フック演出の優先順位づけ。
